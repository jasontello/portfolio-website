Case study / civic-tech redesign

Open Source San José Website Redesign

A homepage evaluation and redesign for a civic-tech organization, focused on preserving the original mission and content while making the hierarchy, spacing, typography, and volunteer path feel more intentional.

Live site ↗ GitHub ↗

Context

The assessment asked me to evaluate an existing homepage, then redesign it with clearer structure and stronger visual execution.

For the BizzNEST Design Associate assessment, I chose the website evaluation and redesign option. The assignment required a written audit of the current Open Source San José homepage and a high-fidelity homepage redesign that addressed issues in visual hierarchy, navigation, mobile responsiveness, CTA clarity, and overall visual cohesion.

Open Source San José already had a strong mission and useful content: volunteers, civic technology, current projects, sponsors, and meeting details. My goal was not to replace that identity. I wanted to keep the organization’s original content and civic-tech purpose, then reorganize the experience so the page felt more polished and easier to scan.

Assignment

I chose Option 2: audit the existing Open Source San José homepage and create a high-fidelity redesign.

The deliverables were a high-fidelity homepage design and a short design rationale explaining the design approach, user experience improvements, challenges, and how the audit informed the redesign.

The hardest part was the content structure. I tried to keep as much of the original information as possible, then improve how that information was grouped and prioritized through typography, spacing, layout rhythm, and clearer section breaks.

Problem

The original homepage had the right information, but the hierarchy made it harder to understand what mattered first.

Visual hierarchy was doing too much at once.

The page included mission text, volunteer CTAs, project cards, sponsors, and location details, but many sections used similar weight and spacing, so the reading path felt less controlled.

The CTA path could be simplified.

The original hero had multiple actions close together. I wanted the redesign to make the main get-involved path feel more deliberate while still keeping donation and community links available.

The content needed more breathing room.

The original site had valuable paragraphs about the organization’s mission and work, but tighter spacing and smaller text made those sections feel dense.

The visual system could feel more software-driven.

Because Open Source San José is a software and civic-tech organization, I wanted the redesign to keep the existing dark, blue, and red brand cues while making the typography and layout feel more technical.

Challenge

Improve the page without making it feel like a different organization.

The main challenge was respecting what was already there. Open Source San José had recognizable colors, a clear volunteer mission, and content that explained the organization well. The redesign needed to make those pieces easier to read instead of replacing them with a completely new brand direction.

I focused on changing the structure around the content: larger type where the page needed emphasis, more consistent spacing, a stronger section rhythm, and a clearer order from mission to projects to sponsors to meeting details.

Design priorities

I used four priorities to keep the redesign focused.

01

Keep the original identity

Carry forward the dark background, bright blue accents, red CTA color, and community photography from the original site.

02

Make it feel technical

Use a code-inspired type direction and sharper spacing so the page immediately gives software-organization cues.

03

Preserve the content

Keep the original mission, project, sponsor, and meeting information, but reorganize it into clearer chunks.

04

Guide the next step

Make volunteering, community details, and donation paths easier to understand without overloading the hero.

User flow

I reorganized the homepage around the order a new visitor would naturally need.

01

Understand the mission

Lead with a direct statement about building technology for a better San José.

02

Scan what the group does

Break Who We Are, Mission, Vision, and What We Do into a tighter row of digestible content blocks.

03

Explore projects

Give project work more visual presence so the organization’s software output feels concrete.

04

Build trust

Keep sponsors and community proof visible, but give them a calmer section layout.

05

Find the next action

Move visitors toward Eventbrite, Discord, directions, volunteering, or donation after the core story is clear.

Content structure

Most of the redesign work was deciding how to keep the original information while making it easier to scan.

I kept the organization’s core sections: who they are, mission, vision, what they do, current projects, sponsors, and where to find them. The redesign changes how those sections are paced and grouped so the visitor does not have to work as hard to understand the page.

The original site used strong brand colors, so I kept the black background, bright blue divider language, and orange-red action color. I made those choices feel more intentional by giving the page a more disciplined grid, larger type contrast, and clearer content modules.

I also chose a type direction that feels closer to code and software culture. That was intentional: I wanted someone landing on the page to immediately feel that this is a technology organization, not just a general nonprofit page.

Insight

The biggest improvement was not inventing new content. It was making the existing content feel intentional.

A civic organization homepage has to do more than look professional. It has to help different visitors orient themselves quickly: a potential volunteer, a sponsor, a community member, or someone simply trying to understand what the organization does.

That insight shaped the redesign around restraint. I kept the original organization’s goals, language, and dominant colors, then used layout and typography to make the same information feel clearer, more modern, and more software-oriented.

Final direction

The final redesign uses a stronger civic-tech identity and a clearer homepage flow.

01

Sharper first screen

The hero keeps the original mission headline and community photography, but gives them more scale and breathing room.

02

Clearer navigation

Top-level links and dropdown behavior support volunteer, about, community, and donation paths without crowding the first screen.

03

Software visual cues

Code-associated typography, dark UI styling, and blue technical accents help the page read as civic tech.

04

Responsive interaction

The page includes mobile navigation, accessible controls, reduced-motion handling, and responsive section stacking.

Build

I built the redesign as a responsive front end after creating the high-fidelity direction in Figma.

The project was implemented with HTML, CSS, and JavaScript and deployed through GitHub Pages. The build focuses on responsive layout, readable hierarchy, accessible navigation, and a visual system that fits the organization’s tech-focused identity.

The final page includes an animated code-rain treatment, mobile menu behavior, dropdown navigation, reduced-motion support, and redesigned homepage sections for projects, sponsors, community context, location, and footer content.

Outcome

The final version keeps Open Source San José recognizable while making the homepage feel more polished and easier to scan.

  • Preserved the organization’s original mission, project, sponsor, and meeting content instead of starting from a blank redesign.
  • Reworked the content structure so a new visitor can move from mission to community proof to action more naturally.
  • Kept the original dominant colors while making the palette feel more controlled and intentional.
  • Used a code-inspired type direction and darker interface style to make the page feel more connected to software and civic tech.

Looking ahead
